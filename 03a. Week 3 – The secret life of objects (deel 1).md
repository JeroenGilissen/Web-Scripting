# Week 3a – The secret life of objects (deel 1)

## Object literals

Een object is een verzameling **key-value pairs**. De syntax is `{ key1: value1, key2: value2, ... }`. Een key is altijd een string of Symbol; een value kan eender welk datatype zijn, inclusief een functie.

```js
let person = {
  name: 'tim',
  age: 7,
  print: function () {
    console.log(`name: ${this.name} age: ${this.age}`);
  }
};

person.print(); // name: tim age: 7
```

> 💡 Wanneer een functie deel uitmaakt van een object, verwijst `this` in die functie naar **het object zelf**.

---

### Geen encapsulation

Object literals hebben geen ingebouwde encapsulation. Conventie: velden die niet buiten het object mogen gewijzigd worden, krijgen een **underscore** als prefix (`_naam`).

```js
let person = {
  _name: 'tim',
  print() {
    console.log(`name: ${this._name}`);
  },
  setName(name) {
    this._name = name;
  },
  getName() {
    return this._name;
  }
};

person.setName('sofie');
person.print();         // name: sofie
person._name = 'jan';   // tegen conventie, maar werkt toch!
person.print();         // name: jan
```

### Shorthand property names (ES2015)

Als de variabelenaam gelijk is aan de gewenste key, mag je de key weglaten:

```js
let name = 'tim';
let age = 7;
const print = function () {
  console.log(`name: ${this.name} age: ${this.age}`);
};

let person = { name, age, print };
// equivalent aan: { name: name, age: age, print: print }
```

### Shorthand method names (ES2015)

Functies als waarde mogen ook zonder het woord `function`:

```js
let person = {
  name: 'tim',
  age: 7,
  print() {
    console.log(`name: ${this.name} age: ${this.age}`);
  }
  // equivalent aan: print: function() { ... }
};
```

## Methods en `this`

Een functie die deel uitmaakt van een object noemen we een **method**. Via `this` heeft de method toegang tot de andere eigenschappen van het object.

```js
let rabbit = {
  type: 'white',
  speak: function (line) {
    console.log(`The ${this.type} rabbit says '${line}'`);
  }
};

rabbit.speak("I'm alive.");
// The white rabbit says 'I'm alive.'
```

## Prototypes (extra)

Elk object is intern verbonden met een **prototype** via een associatierelatie. In het prototype worden gemeenschappelijke functies vastgelegd (bv. `toString` in `Object.prototype`).

Er zijn twee manieren om een object aan te maken met een specifiek prototype:

**`Object.create`** — maak een nieuw object waarvan een bestaand object het prototype wordt:

```js
let protoRabbit = {
  speak(line) {
    console.log(`${this.type} says '${line}'`);
  }
};

let killerRabbit = Object.create(protoRabbit);
killerRabbit.type = "killer";
killerRabbit.speak("SKREEEE!");
// killer says 'SKREEEE!'
```

**`new`** — gebruik een constructor-function om een object te maken. Het prototype van het nieuw object wordt het `.prototype` van de constructor-function:

```js
let Rabbit = function (type) {
  this.type = type;
};

Rabbit.prototype.speak = function (line) {
  console.log(`${this.type} says '${line}'`);
};

Rabbit.prototype.toString = function () {
  return `A ${this.type} rabbit`;
};

let weirdRabbit = new Rabbit("huh?");
weirdRabbit.speak("huh?");       // huh? says huh?
console.log(String(weirdRabbit)); // A huh? rabbit
```

**Relaties in het prototype-systeem:**

| Notatie | Betekenis |
|---|---|
| `[[prototype]]` / `.__proto__` | Relatie tussen objecten (niet-standaard) |
| `.constructor` | Relatie van prototype naar function |
| `.prototype` | Relatie van function naar prototype |

## Classes

Classes bieden een eenvoudigere syntax ten opzichte van prototypes. Intern worden nog altijd prototypes gebruikt.

```js
class Rabbit {
  constructor(type) {
    this.type = type;
  }

  speak(line) {
    console.log(`${this.type} says '${line}'`);
  }

  toString() {
    return `A ${this.type} rabbit`;
  }
}

let killerRabbit = new Rabbit("killer");
let blackRabbit  = new Rabbit("black");
```

### Setters & getters

Met `set` en `get` kun je code uitvoeren telkens een eigenschap wordt gewijzigd of uitgelezen.

```js
class Person {
  constructor(name) {
    this._name = name; // sla op als _name (met underscore!)
  }

  set name(name) {
    this._name = name; // wordt aangeroepen bij: person.name = '...'
  }

  get name() {
    return this._name; // wordt aangeroepen bij: person.name
  }
}

let person = new Person('tim');
person.name = 'sofie';         // setter wordt aangeroepen
console.log(person.name);      // getter wordt aangeroepen → sofie
```

> ⚠️ **Oneindige lus!** Gebruik `_name` als interne opslag, niet `name`. Als je `this.name = name` schrijft in de setter, roept die zichzelf opnieuw aan → stack overflow.

```js
// FOUT: veroorzaakt ∞ loop
set name(name) {
  this.name = name; // roept setter opnieuw aan!
}
```

### Static methods

Een `static` method behoort tot de **klasse zelf**, niet tot een instantie. Je roept hem aan via de klassenaam.

```js
class Temperature {
  constructor(celsius) {
    this.celsius = celsius;
  }

  get fahrenheit() {
    return this.celsius * 1.8 + 32;
  }

  set fahrenheit(value) {
    this.celsius = (value - 32) / 1.8;
  }

  static fromFahrenheit(value) {
    return new Temperature((value - 32) / 1.8);
  }
}

let temp = new Temperature(22);
console.log(temp.fahrenheit);    // → 71.6
temp.fahrenheit = 86;
console.log(temp.celsius);       // → 30

let temp2 = Temperature.fromFahrenheit(86); // via klasse, niet instantie
console.log(temp2.celsius);      // → 30
```

### Inheritance (overerving)

Met `extends` erft een klasse alle methodes van de parent. Met `super` roep je de parent constructor of methode aan.

```js
class Shape {
  constructor(x, y) {
    this.x = x;
    this.y = y;
  }

  describe() {
    console.log(`Shape at (${this.x}, ${this.y})`);
  }

  static describe() {
    console.log('Shape');
  }
}

class Circle extends Shape {
  constructor(x, y, radius) {
    super(x, y);         // roep de Shape constructor aan
    this.radius = radius;
  }

  describe() {
    super.describe();    // roep de Shape describe aan
    console.log('Circle');
    console.log(`My radius is ${this.radius}`);
  }

  static describe() {
    super.describe();
    console.log('Circle');
  }
}

let circle1 = new Circle(4, 5, 6);
circle1.describe();
// Shape at (4, 5)
// Circle
// My radius is 6

Circle.describe();
// Shape
// Circle
```

### instanceof

Met `instanceof` controleer je of een object een instantie is van een bepaalde klasse.

```js
console.log(shape1  instanceof Shape);  // true
console.log(shape1  instanceof Circle); // false
console.log(circle1 instanceof Shape);  // true  (ook de parent!)
console.log(circle1 instanceof Circle); // true
```

> ⚠️ **Veelgemaakte fout:** `!address instanceof Address` werkt **niet** zoals verwacht. De `!` wordt eerst uitgevoerd op `address`, wat `false` of `true` geeft — en `false instanceof Address` is altijd `false`. De correcte syntax is:

```js
// FOUT
if (!address instanceof Address) { ... }

// CORRECT: haakjes rond de instanceof-expressie
if (!(address instanceof Address)) { ... }
```
