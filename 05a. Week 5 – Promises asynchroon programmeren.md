# Web Scripting – Promises: asynchroon programmeren

---

## Callbacks (asynchrone code)

JavaScript is single-threaded maar kan asynchroon werken via callbacks.

```js
// app.js
console.log('start');

setTimeout(() => {
  console.log('waiting 1 sec');
}, 1000);

console.log('stop');
```

> Output:
```
start
stop
waiting 1 sec
```

### Callbacks met Node.js `fs`-module

```js
const fs = require('fs');

fs.readFile('./files/a.txt', 'utf8', (err, data) => {
  if (err) return console.log(err);
  console.log(data);
});

fs.writeFile('./files/a.txt', 'Hello World!', 'utf8', (err) => {
  if (err) return console.log(err);
  console.log('Hello World > ./files/a.txt');
});
```

---

## Callback functions – hoe werkt het?

JavaScript gebruikt een **Call Stack**, **Web APIs**, een **Event Loop** en een **Callback Queue**:

1. Asynchrone functies (bv. `setTimeout`) worden uit de Call Stack gehaald en doorgegeven aan de Web APIs.
2. Wanneer klaar, wordt de callback in de **Callback Queue** geplaatst.
3. De **Event Loop** controleert of de Call Stack leeg is en plaatst dan de callback terug op de stack.

## Promises

Een **Promise** is het resultaat van een uitgestelde en asynchrone bewerking.

- Het retourneert een object waarin het resultaat van de voltooiïng (success) of het mislukken (failure) van de actie terug te vinden is.
- Een promise bevindt zich altijd in één van onderstaande toestanden:

| Toestand | Betekenis |
|----------|-----------|
| **Pending** | Initiële toestand, nog niet fulfilled of rejected |
| **Fulfilled** | Succesvolle uitvoering |
| **Rejected** | Gefaalde uitvoering |
| **Settled** | Ofwel fulfilled of rejected, niet meer pending |

---

## Promises – Syntax

```js
new Promise(executor);
new Promise(function(resolve, reject) { ... });
```

- **Syntax:** inline functie of oproepen promise functie
- Functie object met twee parameters `resolve` en `reject`:
  - `resolve` → vervult (fulfill) de promise
  - `reject` → verwerpt de promise
- Deze functies kunnen worden aangeroepen eens de berekeningen afgerond zijn.
- De functie start **asynchroon**.

---

## Promises – Basisstructuur

```js
let promise = new Promise((resolve, reject) => {
  // doe iets asynchroon
  // bij success:
  //   resolve(somevalue);
  // bij fout:
  //   reject(somevalue);
});

promise
  .then((result) => { console.log(result); })
  .catch((error) => { console.log(error); });
```

---

## Voorbeeld: faculteit van een getal berekenen

```js
function factorial(number) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (typeof number != 'number' || isNaN(number)) {
        return reject(`${number} is not a number`);
      }

      let result = 1;
      for (let i = 1; i <= number; i++) {
        result = result * i;
        if (result == Infinity) {
          break;
        }
      }

      if (result == Infinity) {
        return reject(`${number}! is Infinity`);
      }

      return resolve(result);
    }, Math.floor(Math.random() * 1000));
  });
}

factorial(20)
  .then((result) => { console.log('resolved: ', result); })
  .catch((error) => { console.log('rejected: ', error); });
```

## Chaining

Meerdere `.then()`'s kunnen na elkaar worden gekoppeld.

```js
promise
  .then((result) => { dosomething; return somevalue; })
  .then((result) => { dosomething; return somevalue; })
  .then((result) => { dosomething; return somevalue; })
  .then((result) => { console.log('done: ', result); });
```

### Optie 1: chain aan een niet-asynchrone actie

Het resultaat van een promise wordt gekoppeld aan een gewone (niet-asynchrone) functie.

```js
factorial(100)
  .then((result) => result * 2)
  .then((result) => { console.log('resolved: ', result); })
  .catch((error) => { console.log('rejected: ', error); });
```

> In het voorbeeld is `factorial` de promise. Na de promise wordt verdubbeld en afgedrukt (dit zijn geen promises).

### Optie 2: chain aan een promise

Het resultaat van een promise wordt gekoppeld aan een andere promise.

```js
function double(number) {
  let promise = new Promise((resolve, reject) => {
    setTimeout(() => {
      if (typeof number != 'number' || isNaN(number)) {
        return reject(`${number} is not a number`);
      }
      let result = 2 * number;
      if (result == Infinity) {
        return reject(`${number}! is Infinity`);
      }
      return resolve(result);
    }, Math.floor(Math.random() * 1000));
  });
  return promise;
}

factorial(20)
  .then((result) => double(result))
  .then((result) => { console.log('resolved: ', result); })
  .catch((error) => { console.log('rejected: ', error); });
```

## Promisify

**Promisification** = de conversie van een functie die een callback accepteert naar een functie die een promise teruggeeft.

> Meer info: [https://javascript.info/promisify](https://javascript.info/promisify)

```js
const fs   = require('fs');
const util = require('util');

const readFile = util.promisify(fs.readFile);

readFile('files/a.txt')
  .then((result) => { console.log(result.toString()); })
  .catch((error) => { console.log('rejected: ', error.message); });
```

## async / await

### `async`

- Enkel binnen een `async` function mag `await` gebruikt worden.
- De function geeft **altijd** een Promise terug.
- Als de function geen Promise teruggeeft, wordt de teruggegeven waarde automatisch in een Promise gestoken die de waarde resolve't.

```js
// Deze twee zijn equivalent:
async function f() {       async function f() {
  return 1;           →      return Promise.resolve(1);
}                          }

f().then((result) => { console.log(result); });
```

### `await`

- Wacht op het resultaat van een promise.
- Zorgt voor veel beter leesbare code!
- `await` → vervangt `.then()`

### Error handling met `async`

Als een error wordt opgeworpen in een `async` function, wordt dit vertaald naar een Promise met een `reject`:

```js
// Deze twee zijn equivalent:
async function f() {           async function f() {
  throw new Error("...");  →     return Promise.reject(new Error("..."));
}                              }
```

## async / await – Voorbeeld: factorial herschreven

```js
const util  = require('util');
const sleep = util.promisify(setTimeout);

async function factorial(number) {
  await sleep(Math.floor(Math.random() * 1000));

  if (typeof number != 'number' || isNaN(number)) {
    throw new Error(`${number} is not a number`);
  }

  let result = 1;
  for (let i = 1; i <= number; i++) {
    result = result * i;
    if (result == Infinity) {
      break;
    }
  }

  if (result == Infinity) {
    throw new Error(`${number}! is Infinity`);
  }

  return result;
}

factorial(20)
  .then((result) => { console.log('resolved: ', result); })
  .catch((error) => { console.log('rejected: ', error.message); });
```

## Besluit

**Asynchroon programmeren** laat toe dat JavaScript niet blokkeert tijdens tijdrovende operaties.

**Callbacks** → oudere manier, kan leiden tot "callback hell" bij geneste callbacks.

**Promises** bieden een cleaner alternatief:
- `.then()` → bij succesvolle uitvoering (fulfilled)
- `.catch()` → bij fout (rejected)
- Toestanden: `pending` → `fulfilled` of `rejected` (= `settled`)
- **Chaining**: meerdere `.then()`'s aan elkaar koppelen
- **Promisify**: callback-functie omzetten naar een promise

**async / await** = syntactische suiker bovenop promises:
- `async` function geeft altijd een Promise terug
- `await` wacht op het resultaat van een promise (vervangt `.then()`)
- `throw new Error(...)` wordt automatisch vertaald naar `Promise.reject(...)`
- Resultaat: veel beter leesbare, synchroner uitziende code
