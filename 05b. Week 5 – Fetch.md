## Herhaling JSON

- `JSON.stringify`: vorm een object om naar een JSON-string
- `JSON.parse`: ontleed een JSON-string, maak er een object van

```js
let person = {
  name: "tim",
  hobbies: ["reading", "running", "tennis"]
};

let personJSON = JSON.stringify(person);
console.log(personJSON);
// {"name":"tim","hobbies":["reading","running","tennis"]}

let person2 = JSON.parse(personJSON);
console.log(person2);
// { name: 'tim', hobbies: [ 'reading', 'running', 'tennis' ] }
```

## REST

**Representational State Transfer** – een software-architectuurstijl voor client-server communicatie:

- Software-architectuur **client-server**
- Voorziet standaards tussen computersystemen op het internet om gemakkelijker met elkaar te communiceren
- **Stateless**: elke request bevat alle info die de server nodig heeft
- **Cacheability**: responses kunnen gecached worden
- **Layered system**: client weet niet of het direct met de server praat
- **Uniform interface**: URL's geven data terug (geen HTML)

## Fetch API

De Fetch API laat toe om GET, POST, PUT en DELETE requests te sturen naar een server met een REST-API. De server stuurt een response terug met JSON in de response-body.

### Server opzetten: json-server (Fake REST-API)

1. Installeer npm (normaal al aanwezig via Node.js): [https://www.npmjs.com/get-npm](https://www.npmjs.com/get-npm)

2. Installeer json-server globaal:
```bash
npm install -g json-server
```

3. Maak een `db.json` bestand aan en start de server:
```bash
json-server --watch db.json
```

### HTTP-methodes

| Methode | Omschrijving | Voorbeeld |
|---------|-------------|-----------|
| **GET** | Haal een resource op | `GET /persons/` |
| **POST** | Plaats een nieuwe resource (niet idempotent) | `POST /persons/` + body `{"name":"tim"}` |
| **PUT** | Wijzig een bestaande resource (idempotent) | `PUT /persons/1` + body `{"name":"tim"}` |
| **DELETE** | Verwijder een resource | `DELETE /persons/1` |

> **Idempotent**: één of meerdere keren uitvoeren geeft hetzelfde resultaat.
> **Niet idempotent**: elke uitvoering heeft een ander effect (bv. POST maakt een nieuw record aan).

### Voorbeeld db.json

```json
{
  "persons": [
    { "id": 1, "name": "sofie" },
    { "id": 2, "name": "tim" }
  ]
}
```

### REST-API endpoints

```
GET  http://localhost:3000/persons
GET  http://localhost:3000/persons/1
POST http://localhost:3000/persons   (body: {"name":"geert"})
```

## Fetch: GET

```js
let url    = 'http://localhost:3000/persons/';
let output = document.getElementById("div_output");
makeElementEmpty(output);

fetch(url)                          // verstuur GET request
  .then((response) => {
    if (response.status == 200) {   // status 200 = gelukt
      return response.json();       // parse JSON naar JavaScript
    } else {
      throw `error with status ${response.status}`;
    }
  })
  .then((persons) => {              // vang de array op en doe er iets mee
    let data = [];
    for (let person of persons) {
      data.push([person.id, person.name]);
    }
    console.log(data); // hier kan code om data in de pagina te tonen
  })
  .catch((error) => {               // vang fouten op
    output.appendChild(document.createTextNode(error));
  });
```

## Fetch: POST

```js
let url    = 'http://localhost:3000/persons/';
let output = document.getElementById("div_output");
let name   = document.getElementById("txt_name").value;
let person = { name: name };
makeElementEmpty(output);

fetch(url, {                        // verstuur POST request
  method: "POST",
  body: JSON.stringify(person),     // json in request-body, bv: {"name":"geert"}
  headers: {
    'Accept': 'application/json',
    'Content-Type': 'application/json'
  }
})
  .then((response) => {
    if (response.status == 201) {   // 201 = resource created (gelukt!)
      return response.json();       // parse JSON naar JavaScript
    } else {
      throw `error with status ${response.status}`;
    }
  })
  .then((person) => {               // vang de response op en doe er iets mee
    let data = [];
    data.push([person.id, person.name]);
    console.log(data);
  })
  .catch((error) => {
    output.appendChild(document.createTextNode(error));
  });
```

## Fetch: Chained

Twee fetch-requests na elkaar koppelen. Het resultaat van de eerste request wordt gebruikt als input voor de tweede.

### db2.json

```json
{
  "persons": [
    { "id": 1, "name": "sofie" },
    { "id": 2, "name": "tim" },
    { "id": 3, "name": "tim" },
    { "id": 4, "name": "tim" }
  ]
}
```

### Verloop

1. Eerste fetch: `GET /persons/2` → antwoord: `{ "id": 2, "name": "tim" }`
2. Tweede fetch: `GET /persons/?name=tim` → antwoord: `[{ "id": 2, ... }, { "id": 3, ... }, { "id": 4, ... }]`

### Code

```js
fetch(`${url}${id}`)
  .then((response) => {
    if (response.status == 200) {
      return response.json();
    } else {
      throw `error with status ${response.status}`;
    }
  })
  .then((person) => {
    let name = person.name;
    output.appendChild(document.createTextNode(
      `Id ${id} heeft name ${name}. De name ${name} komt voor op de volgende id's: `
    ));
    return fetch(`${url}?name=${name}`);  // tweede fetch gebaseerd op eerste resultaat
  })
  .then((response) => {
    if (response.status == 200) {
      return response.json();
    } else {
      throw `error with status ${response.status}`;
    }
  })
  .then((persons) => {
    for (let person of persons) {
      let id   = person.id;
      let name = person.name;
      output.appendChild(document.createTextNode(` ${id}`));
    }
  })
```

## Fetch: GET zonder async/await

```js
function fetchPokemonName(id) {
  return fetch(`https://pokeapi.co/api/v2/pokemon/${id}`)
    .then((response) => {
      if (!response.ok) {
        throw new Error(`error status code ${response.status}`);
      }
      return response.json();
    })
    .then((pokemon) => pokemon.name);
}

for (let i = 0; i < 10; i++) {
  fetchPokemonName(i)
    .then((name)  => { console.log(name); })
    .catch((error) => { console.log(error.message); });
}
```

> Output (asynchroon, volgorde niet gegarandeerd):
 ```
error status code 404
ivysaur
bulbasaur
venusaur
charmander
...
 ```

## Fetch: GET met async/await

Zelfde functionaliteit, maar veel leesbaarder met `async/await`:

```js
async function fetchPokemonName(id) {
  let response = await fetch(`https://pokeapi.co/api/v2/pokemon/${id}`);
  if (!response.ok) {
    throw new Error(`error status code ${response.status}`);
  }
  let pokemon = await response.json();
  return pokemon.name;
}

for (let i = 0; i < 10; i++) {
  fetchPokemonName(i)
    .then((name)  => { console.log(name); })
    .catch((error) => { console.log(error.message); });
}
```

## Fetch: GET op ID met async/await

```js
// url bepalen en ophalen nodes uit html pagina
let url     = 'http://localhost:3000/persons/';
let idInput = document.getElementById('id');
let output  = document.getElementById("div_output");
makeElementEmpty(output);

// de functie om de gegevens van 1 persoon met id op te halen
async function fetchPerson(id) {
  let response = await fetch(url + id);
  if (!response.ok) {
    throw `error with status ${response.status}`;
  }
  let person = await response.json();
  return person;
}

// functie koppelen na klikken op enter in input
// id uit inputveld wordt opgehaald en de functie fetchPerson wordt opgeroepen
// er wordt gewacht op de teruggegeven waardes en deze worden weergegeven
idInput.addEventListener('keypress', (event) => {
  if (event.key === "Enter") {
    event.preventDefault();
    makeElementEmpty(output);
    let idwaarde = (idInput.value).trim();

    fetchPerson(idwaarde)
      .then((person) => {
        output.appendChild(document.createTextNode(person.name));
      })
      .catch((error) => {
        console.log(error.message);
      });
  }
});
```

## Hulpfunctie: makeElementEmpty

```js
function makeElementEmpty(element) {
  while (element.hasChildNodes()) {
    element.removeChild(element.firstChild);
  }
}
```

> Verwijdert alle child-nodes van een element. Wordt gebruikt om de output-div leeg te maken voor nieuwe data wordt getoond.

## Besluit

**REST** is een architectuurstijl waarbij URL's data teruggeven via standaard HTTP-methodes (GET, POST, PUT, DELETE).

**Fetch API** maakt het mogelijk om vanuit JavaScript HTTP-requests te sturen naar een REST-server en JSON-responses te verwerken.

**Twee stijlen** voor fetch-requests:
- **Zonder async/await**: via `.then()` chaining — werkt, maar kan complex worden
- **Met async/await**: veel leesbaarder, ziet eruit als synchrone code

**HTTP statuscodes** bij fetch zijn belangrijk:
- `200` → GET gelukt
- `201` → POST gelukt (resource created)
- Andere codes → fout, gooi een error op via `.catch()`

**json-server** is een handige tool om snel een nep REST-API op te zetten tijdens ontwikkeling, op basis van een eenvoudig `db.json` bestand.
