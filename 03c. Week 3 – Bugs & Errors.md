# Week 3c – Bugs & Errors

## `use strict`

Zet `'use strict'` bovenaan een bestand of in een functie om stricter gedrag te activeren. Het voornaamste nut: **global bindings vermijden**.

### Probleem zonder `use strict`

Als je vergeet `let`, `const` of `var` te schrijven, maakt JavaScript automatisch een **globale variabele** aan in het global object (`window` in de browser, `global` in Node.js):

```js
// ZONDER use strict
name = 'tim';          // vergeten: let name = 'tim';
console.log(name);         // tim
console.log(global.name);  // tim  ← in het global object terechtgekomen!
```

### Met `use strict`

```js
'use strict'
name = 'tim'; // foutmelding!
// ReferenceError: name is not defined
```

### `use strict` in een functie

```js
function canYouSpotTheProblem() {
  'use strict';
  for (counter = 0; counter < 10; counter++) { // let vergeten!
    console.log("Happy happy");
  }
}

canYouSpotTheProblem();
// ReferenceError: counter is not defined
```

### Effect op `this`

`use strict` verandert ook het gedrag van `this` buiten objecten:

| Situatie | Zonder `use strict` | Met `use strict` |
|---|---|---|
| Functie deel van object | `this` = het object | `this` = het object |
| Losstaande functie | `this` = global object | `this` = `undefined` |

**Praktisch voorbeeld:** als je `new` vergeet bij het aanroepen van een constructor-function:

```js
// ZONDER use strict
function Person(name) {
  this.name = name;
}
let ferdinand = Person("Ferdinand"); // new vergeten!
// Person wordt aangeroepen als gewone functie → this = global object
console.log(name); // Ferdinand ← nu een globale variabele!

// MET use strict
'use strict';
function Person(name) {
  this.name = name;
}
let ferdinand = Person("Ferdinand"); // new vergeten!
// TypeError: Cannot set property 'name' of undefined
// → this is undefined, fout wordt direct gesignaleerd
```

> ✅ `use strict` vangt vergeten `new` op, wat zonder strict mode een stille fout veroorzaakt.

## `try … catch`

Met `try...catch` kan je fouten **opvangen** zodat het programma niet crasht.

```js
try {
  // code die mogelijk een fout gooit
  // bij een fout: executie stopt hier, spring naar catch
} catch (error) {
  // wordt uitgevoerd als er een error opgeworpen werd
  // error.message bevat de foutboodschap
}
```

Met `finally` (optioneel): wordt **altijd** uitgevoerd, ongeacht of er een fout was:

```js
try {
  // risicovolle code
} catch (error) {
  // foutafhandeling
} finally {
  // wordt altijd uitgevoerd (ook na return of throw in try/catch)
}
```

## `throw`

Met `throw` gooi je zelf een fout op. De verdere uitvoering van de functie stopt onmiddellijk.

```js
'use strict';

function invert(number) {
  if (typeof number != 'number') {
    throw new Error("not a number");
  }
  if (number == 0) {
    throw new Error("division by 0");
  }
  return 1 / number;
}

try {
  let result = invert(0);
  console.log(`result = ${result}`);
} catch (error) {
  console.log(`Exception: ${error.message}`);
}
// Exception: division by 0
```

## Specifieke errors opvangen

Door eigen error-klassen te maken die `Error` uitbreiden, kun je in de catch block **onderscheid maken** tussen verschillende soorten fouten.

```js
'use strict';

class InputError extends Error {}
class ArithmeticError extends Error {}

function invert(number) {
  if (typeof number != 'number') {
    throw new InputError("not a number");
  }
  if (number == 0) {
    throw new ArithmeticError("division by 0");
  }
  return 1 / number;
}

try {
  let result = invert(0);
  console.log(`result = ${result}`);
} catch (error) {
  if (error instanceof InputError) {
    console.log(`InputError: ${error.message}`);
  } else if (error instanceof ArithmeticError) {
    console.log(`ArithmeticError: ${error.message}`);
  }
}
// ArithmeticError: division by 0
```

> 💡 Gebruik `instanceof` in de catch block om te bepalen welk type fout er opgeworpen werd.

## Overzicht

| Concept | Sleutelwoord | Nut |
|---|---|---|
| Strict modus | `'use strict'` | Voorkomt global bindings en stille fouten |
| Fout proberen op te vangen | `try { }` | Omhult risicovolle code |
| Fout afhandelen | `catch (error) { }` | Reageert op een opgeworpen fout |
| Altijd uitvoeren | `finally { }` | Opruimcode die altijd loopt |
| Fout opgooien | `throw new Error(...)` | Stopt executie, geeft foutboodschap mee |
| Eigen fouttype | `class MyError extends Error {}` | Maakt specifieke afhandeling mogelijk |
