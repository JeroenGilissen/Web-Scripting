# Web Scripting – Modules

## Wat zijn modules?

- Opsplitsing van code in **files / directories**
- **Dependencies**: code is afhankelijk van andere code
- **npm** (node package manager): cli-tool die externe dependencies installeert

Twee systemen:
- **CommonJS modules** (1tin) → `module.exports` + `require`
- **ECMAScript modules** (2tin) → `import` / `export`

---

## CommonJS Modules

### Exporteren via `module.exports`

`module.exports` is het object dat vanuit een module geëxporteerd wordt. Standaard is dit een lege object literal `{}`.

**Manier 1 – Velden toevoegen aan het exports-object**

```js
// text.js
'use strict'
module.exports.uppercase = (text) => text.toUpperCase();
module.exports.lowercase = (text) => text.toLowerCase();
```

```js
// app.js
'use strict'
const textUtils = require('./text'); // importeer de module
let result = textUtils.uppercase('test');
console.log(result); // TEST
```

> `require('./text')` importeert `text.js` uit dezelfde directory.

---

**Manier 1b – Importeren met destructuring**

```js
// app.js
'use strict'
const { uppercase, lowercase } = require('./text');
let result = uppercase('test');
console.log(result); // TEST
```

---

**Manier 2 – Nieuw object toekennen aan `module.exports` (shorthand notation)**

```js
// text.js
const uppercase = (text) => text.toUpperCase();
const lowercase = (text) => text.toLowerCase();

module.exports = { lowercase, uppercase }; // shorthand notation
```

```js
// app.js
const { uppercase, lowercase } = require('./text'); // destructuring
console.log(uppercase('Ok')); // OK
console.log(lowercase('Ok')); // ok
```

---

**Manier 3 – `module.exports` is een functie (factory pattern)**

```js
// util/printer.js
module.exports = function(text) {
    return {
        print: function() {
            console.log(text); // text bewaard in closure
        },
        printUppercase: function() {
            console.log(text.toUpperCase());
        }
    }
};
```

```js
// app.js
let printer = require('./util/printer.js')('tim');
printer.printUppercase(); // TIM
```

> De module exporteert een **functie** die je direct aanroept met een argument. De `text`-parameter wordt bewaard in een **closure**.

---

**Manier 4 – `module.exports` is een class**

```js
// util/Point.js
module.exports = class Point {
    constructor(x, y) {
        if (typeof x != 'number' || typeof y != 'number') {
            throw new Error("Parameter is not a number!");
        }
        this._x = x;
        this._y = y;
    }

    get x() { return this._x; }
    get y() { return this._y; }

    toString() {
        return `(${this._x},${this._y})`;
    }
}
```

```js
// app.js
const Point = require('./util/Point');
let p = new Point(1, 2);
console.log(p.toString()); // (1,2)
```

## Externe dependencies via npm

```bash
npm install simple-node-logger
# of korter:
npm i simple-node-logger
```

Na installatie staat de dependency in de map `node_modules/`.

```js
// app.js
const log = require('simple-node-logger').createSimpleLogger('project.log');
log.info('subscription accepted at ', new Date().toJSON());
```

Na uitvoering wordt `project.log` aangemaakt:
```
09:54:34.696 INFO  subscription accepted at 2021-09-10T07:54:34.696Z
```

> Bij externe packages gebruik je de **packagenaam** in `require()`, geen relatief pad.

---

## Samenvatting

| Concept | Wat |
|---|---|
| `module.exports` | Object (of functie/class) dat je exporteert |
| `require('./pad')` | Importeer een lokale module |
| `require('pakket')` | Importeer een npm-pakket |
| Destructuring | `const { a, b } = require(...)` |
| Shorthand notation | `module.exports = { a, b }` |
| npm | Installeert externe packages in `node_modules/` |
