# JavaScript Functies

## Function Definities

Er zijn drie manieren om functies te definiëren in JavaScript, elk met hun eigen karakteristieken en gebruik.

### 1. Definitie via Binding (Function Expression)

Functies worden toegekend aan variabelen. De functie is alleen beschikbaar **na** de definitie.

```javascript
// Met const (aanbevolen - niet wijzigbaar)
const square = function(x) {
    return x * x;
};

console.log(square(2)); // 4
```

```javascript
// Met let (wijzigbaar - kan handig zijn voor conditionele functies)
let safeMode = true;
let launchMissiles = function() {
    console.log("launching missiles");
};

if (safeMode) {
    launchMissiles = function() {
        console.log("nothing happens");
    };
}

launchMissiles(); // "nothing happens"
```

### 2. Definitie via Declaration (Function Declaration)

Function declarations worden **gehoisted** - je mag ze aanroepen vóór de declaratie.

```javascript
// Dit werkt door hoisting
future(); // "You'll never have flying cars"

function future() {
    console.log("You'll never have flying cars");
}

future(); // "You'll never have flying cars"
```

**Let op:** Function expressions worden NIET gehoisted:

```javascript
// Dit geeft een ReferenceError
future(); // ReferenceError: Cannot access 'future' before initialization

const future = function() {
    console.log("You'll never have flying cars");
};
```

### 3. Definitie via Arrow Functions (ES6)

Arrow functions bieden een kortere syntax en hebben andere `this`-binding eigenschappen.

```javascript
// Uitgebreide vorm
const power = (base, exponent) => {
    let result = 1;
    for (let i = 0; i < exponent; i++) {
        result *= base;
    }
    return result;
};

console.log(power(2, 3)); // 8

// Kortere vormen
const square = (x) => { return x * x; };
console.log(square(2)); // 4

// Één parameter: haakjes mogen weg
const square2 = x => { return x * x; };

// Zonder accolades: automatische return van expression
const square3 = x => x * x;
const add = (a, b) => a + b;
```

## Optional Arguments

JavaScript is flexibel met functie-argumenten.

### Extra Argumenten Worden Genegeerd

```javascript
function square(x) {
    return x * x;
}

console.log(square(2, "a", 12223)); // 4 - extra argumenten worden genegeerd
```

### Ontbrekende Argumenten Worden `undefined`

```javascript
function minus(a, b) {
    if (b == undefined) {
        return -a;  // Unary minus
    } else {
        return a - b;
    }
}

console.log(minus(3));    // -3 
console.log(minus(3, 4)); // -1
```

### Default Parameters (ES6)

```javascript
function minus(a, b = 0) {
    return a - b;
}

console.log(minus(5));    // 5
console.log(minus(5, 2)); // 3
```

## Scope van Variabelen

### var (Vermijd dit - Legacy)

- **Function scope** binnen functies, **global scope** buiten functies
- **Hoisting**: declaraties worden naar boven gebracht, initialisaties niet

```javascript
// Wat je schrijft:
console.log(a); // undefined (niet ReferenceError!)
var a = 1;
console.log(a); // 1

// Hoe JavaScript het interpreteert:
var a;           // Hoisting - declaratie naar boven
console.log(a);  // undefined
a = 1;           // Initialisatie blijft op originele plaats
console.log(a);  // 1
```

**Voorbeelden van verwarrend var gedrag:**

```javascript
var x = 1;
function test() {
    console.log(x); // 1 - gebruikt global x
}
test();

// Maar met var binnen de functie:
var x = 1;
function test() {
    console.log(x); // undefined! (niet 1)
    if (1 == 2) {   // Deze code wordt nooit uitgevoerd
        var x = -1; // Maar var x wordt wel gehoisted!
    }
}
test(); // undefined
```

### let/const (Modern - Gebruik Deze)

- **Block scope** - alleen binnen `{}` blokken waar ze gedeclareerd zijn
- **Hoisting** met **Temporal Dead Zone (TDZ)** - ReferenceError bij gebruik voor declaratie

```javascript
// TDZ Voorbeeld - FOUT
console.log(a); // ReferenceError: Cannot access 'a' before initialization
let a;
console.log(a); // undefined
a = 1;
console.log(a); // 1

// Correct gebruik
let a;
console.log(a); // undefined
a = 1;  
console.log(a); // 1
```

### Variable Shadowing Voorbeeld

```javascript
let aantal = 10;
for (let i = 0; i < aantal; i++) {
    let aantal = parseInt(5 * Math.random()); // Lokale 'aantal' 
    console.log("*".repeat(aantal)); // Gebruikt lokale 'aantal' (0-4)
}
// Loop draait 10 keer (outer aantal), print 0-4 sterretjes (inner aantal)
```

**TDZ Fout Voorbeeld:**

```javascript
let aantal = 10;
for (let i = 0; i < aantal; i++) {
    console.log(aantal); // ReferenceError! TDZ
    let aantal = parseInt(5 * Math.random()); 
    console.log("*".repeat(aantal));
}
```

**Correct - Geen Shadowing:**

```javascript
let aantal = 10;
for (let i = 0; i < aantal; i++) {
    console.log(aantal);  // 10 (gebruikt outer scope)
    aantal = parseInt(5 * Math.random()); // Wijzigt outer scope variabele
    console.log("*".repeat(aantal));
}
```

## Nested Scope

Functies kunnen binnen andere functies gedefinieerd worden en hebben toegang tot outer scope variabelen.

```javascript
const hummus = function(factor) {
    const ingredient = function(amount, unit, name) {
        let ingredientAmount = amount * factor; // Gebruikt 'factor' van outer scope
        if (ingredientAmount > 1) {
            unit += "s";
        }
        console.log(`${ingredientAmount} ${unit} ${name}`);
    };
    
    ingredient(1, "can", "chickpeas");
    ingredient(0.25, "cup", "tahini");
    ingredient(0.25, "cup", "lemon juice");
};

hummus(3);
// Output:
// 3 cans chickpeas
// 0.75 cups tahini  
// 0.75 cups lemon juice
```

**Scope Hiërarchie:**

- `hummus` functie heeft: `factor`, `ingredient`
- `ingredient` functie heeft: `amount`, `unit`, `name`, `ingredientAmount` + `factor` (van outer scope)

## Closures

Een closure ontstaat wanneer een inner function toegang heeft tot variabelen van een outer function, zelfs nadat de outer function is uitgevoerd.

```javascript
const generateMultiplier = function(factor) {
    return function(number) {
        return number * factor; // 'factor' blijft beschikbaar!
    };
};

const twice = generateMultiplier(2);
const threeTimes = generateMultiplier(3);

console.log(twice(5));      // 10
console.log(threeTimes(6)); // 18

// De returned functions "onthouden" hun factor waarde
```

**Praktisch Voorbeeld:**

```javascript
const createCounter = function(start = 0) {
    let count = start;
    
    return {
        increment: () => ++count,
        decrement: () => --count,
        getValue: () => count,
        reset: () => count = start
    };
};

const counter = createCounter(10);
console.log(counter.getValue()); // 10
console.log(counter.increment()); // 11
console.log(counter.increment()); // 12
console.log(counter.decrement()); // 11
counter.reset();
console.log(counter.getValue()); // 10
```

## Recursie

Recursie is een techniek waarbij een functie zichzelf aanroept om een probleem op te lossen.

### Basis Recursie - Machtsverheffing

```javascript
function power(base, exponent) {
    if (exponent == 0) {
        return 1; // Base case
    }
    return base * power(base, exponent - 1); // Recursive case
}

console.log(power(2, 3)); // 8
// Uitvoering: 2 * power(2, 2) 
//            2 * 2 * power(2, 1)
//            2 * 2 * 2 * power(2, 0)
//            2 * 2 * 2 * 1 = 8
```

### Faculteit Berekening

```javascript
function factorial(number) {
    if (number == 0) {
        return 1; // 0! = 1
    }
    return number * factorial(number - 1);
}

console.log(factorial(5)); // 120
// 5! = 5 * 4 * 3 * 2 * 1 = 120
```

### Complexere Recursie - Pad Zoeken

```javascript
function findSolution(target) {
    function find(current, history) {
        console.log(`find: ${current} = ${history}`);
        
        if (current == target) {
            return history;
        } else if (current > target) {
            return null;
        } else {
            // Probeer beide operaties: +5 of *3
            return find(current + 5, `(${history} + 5)`) ||
                   find(current * 3, `(${history} * 3)`);
        }
    }
    
    return find(1, "1");
}

console.log(findSolution(13));
// Output toont alle pogingen, eindresultaat: "((1 * 3) + 5) + 5"
```

## Higher-Order Functions

Functies die andere functies als argument nemen of teruggeven.

```javascript
// Functie als argument
function repeat(n, action) {
    for (let i = 0; i < n; i++) {
        action(i);
    }
}

repeat(3, console.log);
// 0
// 1  
// 2

// Met arrow function
repeat(3, i => console.log(`Number: ${i}`));
// Number: 0
// Number: 1
// Number: 2
```

## Array Methods met Functies

```javascript
const numbers = [1, 2, 3, 4, 5];

// map - transform elk element
const doubled = numbers.map(x => x * 2);
console.log(doubled); // [2, 4, 6, 8, 10]

// filter - selecteer elementen
const evens = numbers.filter(x => x % 2 === 0);
console.log(evens); // [2, 4]

// reduce - combineer tot één waarde
const sum = numbers.reduce((total, x) => total + x, 0);
console.log(sum); // 15

// forEach - voer actie uit voor elk element
numbers.forEach(x => console.log(`Value: ${x}`));
```

## Samenvatting

### Function Declaratie Manieren

```javascript
// 1. Function Expression (const/let + function)
const twice = function(number) {
    return number * 2;
};

// 2. Function Declaration (hoisted)
function threeTimes(number) {
    return number * 3;
}

// 3. Arrow Function (kort + andere this-binding)
const fourTimes = number => number * 4;
const fiveTimes = (number) => { return number * 5; };
```

### Belangrijke Concepten

**Scope:**

- `var` = function scope + hoisting (vermijd)
- `let`/`const` = block scope + TDZ (gebruik deze)

**Closures:**

- Inner functions hebben toegang tot outer function variabelen
- Variabelen blijven beschikbaar zelfs na afloop outer function

**Recursie:**

- Functie roept zichzelf aan
- Moet base case hebben om oneindige loop te voorkomen
- Handig voor boom-structuren, wiskundige berekeningen

**Best Practices:**

- Gebruik `const` voor functies die niet veranderen
- Gebruik `let` alleen als je de functie moet wijzigen
- Gebruik arrow functions voor korte callback functions
- Geef functies duidelijke, beschrijvende namen
- Houd functies klein en gefocust op één taak
