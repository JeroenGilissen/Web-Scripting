# Web Scripting – Javascript and the Browser

## HTTP-protocol

De browser communiceert met een webserver via het HTTP-protocol:

1. Gebruiker klikt op een link of typt een URL
2. Browser stuurt een **HTTP Request** naar de server
3. Server verwerkt de aanvraag en stuurt een **HTTP Response** terug
4. Browser toont het resultaat

**HTTP Request** bestaat uit:
- **HTTP Method** (GET, POST, PUT, DELETE, ...)
- **Path** (bijv. `/hello`)
- **HTTP Version** (bijv. `HTTP/1.1`)
- **Request body** (optioneel, bijv. bij POST)

**HTTP Response** bestaat uit:
- **HTTP Version** + **Status code** (bijv. `HTTP/1.1 200 OK`)
- **Response body** (bijv. de HTML-pagina)

## index.html: CSS, img, JS

Een HTML-pagina kan verwijzen naar externe bestanden. De browser laadt deze apart op via afzonderlijke HTTP requests.

```html
<!-- index.html -->
<!doctype html>
<html>
    <head>
        <meta charset="UTF-8">
        <title>index.html</title>
        <link href="/assets/css/mainstyle.css" rel="stylesheet" type="text/css" />
    </head>
    <body>
        <img src="/assets/images/pxl_it.png" alt="PXL logo" />
        <script src="/assets/js/hello.js"></script>
    </body>
</html>
```

```css
/* assets/css/mainstyle.css */
img {
    width: 200px;
}
```

```js
// assets/js/hello.js
alert("hello");
```

> In de browser DevTools (Ctrl+Shift+I → Network) zie je alle afzonderlijke requests: `index.html`, het CSS-bestand, de afbeelding, het JS-bestand, ...

---

## HTTP Headers inspecteren

Via DevTools → Network → klik op een bestand → Headers:

- **Request URL**: het volledige adres
- **Request Method**: GET, POST, ...
- **Status Code**: 200 OK = succes
- **Response Headers**: info van de server (Content-Type, Content-Length, ...)
- **Request Headers**: info van de browser (User-Agent, Accept-Language, ...)

## HTTP Methods: GET, POST, PUT, DELETE

REST is een **client-server architectuur** waarbij resources via HTTP worden aangesproken.

**GET** – haal een resource op
```
GET /persons/
```

**POST** – maak een nieuwe resource aan
```
POST /persons/
body: {"name": "tim"}
```
> ⚠️ **Niet idempotent**: elke keer dat je POST uitvoert, wordt er een nieuw record aangemaakt (id=1, id=2, ...).

**PUT** – maak aan of wijzig een bestaande resource
```
PUT /persons/1
body: {"name": "tim"}
```
> ✅ **Idempotent**: één of meerdere keren uitvoeren geeft hetzelfde resultaat (id=1, name="tim").

**DELETE** – verwijder een resource
```
DELETE /persons/1
```

## Javascript & HTML

### Inline script

JS-code kan rechtstreeks in HTML worden geplaatst via de `<script>`-tag. Een extern `.js` bestand is aangeraden.

```html
<!-- index.html -->
<!doctype html>
<html>
    <head>
        <meta charset="UTF-8">
        <title>index.html</title>
    </head>
    <body>
        <script>alert("hallo");</script>
    </body>
</html>
```

### onclick attribuut

JS kan ook direct op een HTML-element gekoppeld worden via `onclick`. Later leer je dit netter met W3C eventhandling.

```html
<!-- index.html -->
<!doctype html>
<html>
    <head>
        <meta charset="UTF-8">
        <title>index.html</title>
    </head>
    <body>
        <button onclick="alert('Boom!');">DO NOT PRESS</button>
    </body>
</html>
```

## Samenvatting

| Concept | Wat |
|---|---|
| HTTP Request | Browser vraagt resource op bij server |
| HTTP Response | Server antwoordt met status + body |
| GET | Resource ophalen |
| POST | Nieuwe resource aanmaken (niet idempotent) |
| PUT | Resource aanmaken of wijzigen (idempotent) |
| DELETE | Resource verwijderen |
| `<script src="...">` | Extern JS-bestand inladen |
| `<script>...</script>` | Inline JS in HTML |
| `onclick="..."` | JS koppelen aan een HTML-element |
| DevTools Network | HTTP requests & responses inspecteren |
